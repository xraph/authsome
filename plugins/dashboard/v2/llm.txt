# Dashboard Plugin

## Purpose
Server-rendered admin interface built with Alpine.js and Tailwind CSS. Provides web-based management for users, organizations, roles, API keys, and audit logs.

## Features
- **Authentication**: Built-in auth with session management
- **RBAC**: Permission-based access control
- **CSRF Protection**: Session-bound tokens
- **Dark Mode**: System preference detection
- **Real-time Stats**: User counts, active sessions
- **User Management**: List, view, edit, ban users
- **Organization Management**: Orgs, members, teams
- **API Key Management**: Create, revoke, view keys
- **Audit Log Viewer**: Security event timeline
- **Settings**: System configuration
- **Extensions**: Plugin extension system

## Routes
- `GET /dashboard/` - Home page
- `GET /dashboard/users` - User list
- `GET /dashboard/organizations` - Org list
- `GET /dashboard/api-keys` - API key list
- `GET /dashboard/audit` - Audit logs
- `GET /dashboard/settings` - Settings

## Configuration
```yaml
auth:
  plugins:
    dashboard:
      enabled: true
      path: "/ui"
      requireAdmin: true
```

## Tech Stack
- Go templates (server-side rendering)
- Alpine.js (~15KB)
- Tailwind CSS 4 CDN
- HTMX for dynamic updates

## Extension API
```go
type Extension interface {
    ID() string
    MenuItems() []MenuItem
    Pages() []Page
    Widgets() []Widget
}
```

## Dependencies
- **core/user**: User management
- **core/organization**: Org management
- **core/rbac**: Permission checking
- **core/audit**: Audit log queries
- **core/apikey**: API key management

## Used By
Other plugins extend dashboard with custom pages and widgets.

## Security
- Admin role required by default
- CSRF protection on all forms
- Permission checks on every action
- Rate limiting on endpoints
- Session-based authentication
