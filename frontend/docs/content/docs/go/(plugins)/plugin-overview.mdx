## Plugin Architecture

AuthSome Go plugins follow a consistent architecture pattern that makes them easy to develop, configure, and integrate:

```go
// Plugin interface that all plugins must implement
type Plugin interface {
    ID() string                                    // Unique plugin identifier
    Init(auth *authsome.Auth) error               // Initialize plugin with dependencies
    RegisterRoutes(router forge.Router) error     // Register HTTP routes
    RegisterHooks(hooks *HookRegistry) error      // Register lifecycle hooks
    Migrate() error                               // Run database migrations
}
```

    ```go
    import (
        "github.com/xraph/authsome"
        "github.com/xraph/authsome/plugins/username"
        "github.com/xraph/authsome/plugins/twofa"
        "github.com/xraph/authsome/plugins/social"
        "github.com/xraph/authsome/plugins/passkey"
        "github.com/xraph/authsome/plugins/sso"
        "github.com/xraph/authsome/plugins/admin"
    )

    auth, err := authsome.New(config)
    if err != nil {
        log.Fatal(err)
    }

    // Register core plugins
    auth.RegisterPlugin(username.NewPlugin())
    auth.RegisterPlugin(twofa.NewPlugin())
    auth.RegisterPlugin(anonymous.NewPlugin())

    // Register advanced plugins
    auth.RegisterPlugin(social.NewPlugin())
    auth.RegisterPlugin(passkey.NewPlugin())
    auth.RegisterPlugin(magiclink.NewPlugin())

    // Register enterprise plugins
    auth.RegisterPlugin(sso.NewPlugin())
    auth.RegisterPlugin(admin.NewPlugin())

    // Initialize all plugins
    if err := auth.InitializePlugins(); err != nil {
        log.Fatal(err)
    }

    // Mount to Forge app
    app := forge.New()
    auth.Mount(app, "/auth")
    app.Listen(":8080")
    ```

## Plugin Hooks

Plugins can register hooks to extend functionality at various points in the authentication flow:

```go
// Available hook points
const (
    // User lifecycle
    BeforeUserCreate = "before:user:create"
    AfterUserCreate  = "after:user:create"
    BeforeUserUpdate = "before:user:update"
    AfterUserUpdate  = "after:user:update"
    BeforeUserDelete = "before:user:delete"
    AfterUserDelete  = "after:user:delete"

    // Authentication
    BeforeSignUp = "before:signup"
    AfterSignUp  = "after:signup"
    BeforeSignIn = "before:signin"
    AfterSignIn  = "after:signin"
    BeforeSignOut = "before:signout"
    AfterSignOut  = "after:signout"

    // Session management
    BeforeSessionCreate = "before:session:create"
    AfterSessionCreate  = "after:session:create"
    BeforeSessionRevoke = "before:session:revoke"
    AfterSessionRevoke  = "after:session:revoke"

    // Password management
    BeforePasswordChange = "before:password:change"
    AfterPasswordChange  = "after:password:change"
    BeforePasswordReset  = "before:password:reset"
    AfterPasswordReset   = "after:password:reset"
)
```

## Plugin Development

Create custom plugins by implementing the Plugin interface:

```go
package myplugin

import (
    "github.com/xraph/authsome"
    "github.com/xraph/forge"
)

type MyCustomPlugin struct {
    config  MyPluginConfig
    service *MyPluginService
}

type MyPluginConfig struct {
    Enabled bool   `json:"enabled"`
    APIKey  string `json:"apiKey"`
    Timeout int    `json:"timeout"`
}

func NewPlugin() *MyCustomPlugin {
    return &MyCustomPlugin{}
}

func (p *MyCustomPlugin) ID() string {
    return "my-custom-plugin"
}

func (p *MyCustomPlugin) Init(auth *authsome.Auth) error {
    if err := auth.Config().Bind("auth.plugins.myCustomPlugin", &p.config); err != nil {
        return err
    }

    p.service = NewMyPluginService(p.config)
    return nil
}

func (p *MyCustomPlugin) RegisterRoutes(router forge.Router) error {
    handler := NewMyPluginHandler(p.service)
    router.POST("/my-custom/action", handler.CustomAction)
    router.GET("/my-custom/status", handler.GetStatus)
    return nil
}

func (p *MyCustomPlugin) RegisterHooks(hooks *HookRegistry) error {
    hooks.Register(AfterSignIn, p.onSignIn)
    return nil
}

func (p *MyCustomPlugin) Migrate() error {
    return p.service.Migrate()
}
```

<Callout type="info">
**Plugin Best Practices**: Always handle errors gracefully, use proper configuration binding, implement comprehensive logging, and follow the established patterns for consistency.
</Callout>

## Available Plugins

### Core Plugins
Essential authentication methods that most applications need:

- **Email & Password** - Traditional email/password authentication with sign up, sign in, password reset
- **Two-Factor Authentication** - TOTP, backup codes, and trusted devices
- **Username Authentication** - Username/password login with account lockout and password history
- **Anonymous Authentication** - Guest user sessions with account linking
- **Email Verification** - Email address verification with auto-send on signup
- **Bearer Token** - Bearer token authentication middleware for session-based auth
- **Dashboard** - Server-rendered admin dashboard UI with plugin extensions

### Advanced Plugins
Enhanced features for better security and user experience:

- **Social OAuth** - Google, GitHub, Facebook, Apple, Microsoft, Discord, and 10+ more providers
- **Passkey/WebAuthn** - Modern biometric and hardware key authentication
- **Magic Link** - Passwordless email-based authentication
- **Email OTP** - One-time password via email
- **Phone Authentication** - SMS-based verification and authentication
- **Multi-Factor Authentication** - Advanced MFA with risk-based adaptive authentication
- **MCP** - Model Context Protocol integration for AI-powered workflows
- **JWT** - JWT token generation, verification, and JWKS endpoint
- **API Key** - API key authentication with rate limiting, rotation, and IP restrictions
- **Multi-Session** - Multiple concurrent session management with device tracking
- **Impersonation** - Secure admin-to-user impersonation with audit logging
- **Notification** - Email/SMS template management and delivery with multiple providers

### Enterprise Plugins
Enterprise-grade features for large organizations:

- **Admin API** - User management, session management, role assignment, and system stats
- **SSO** - Enterprise Single Sign-On with SAML 2.0 and OIDC
- **OIDC Provider** - Full OpenID Connect Provider with OAuth 2.0 authorization
- **Multi-Tenancy** - Enterprise multi-tenant organization management
- **Multi-App Platform** - Platform-level app management with environments
- **Organizations** - User-created organizations with teams and invitations
- **Subscription & Billing** - SaaS subscription management with Stripe integration
- **Secrets Management** - Encrypted secrets with versioning and Forge ConfigSource
- **CMS** - Headless CMS with content types, revisions, and draft/publish workflow
- **SCIM 2.0** - Automated user provisioning from enterprise identity providers
- **Backup Authentication** - Account recovery with multiple verification methods
- **Compliance** - SOC 2, HIPAA, PCI-DSS, GDPR, and ISO 27001 compliance
- **Consent Management** - GDPR/CCPA consent, cookie consent, and data portability
- **Geographic Fencing** - Location-based access control with VPN/proxy detection
- **Identity Verification** - KYC with Onfido, Jumio, and Stripe Identity
- **mTLS** - Mutual TLS with X.509 certificates and smart card support
- **Step-Up Authentication** - Context-aware re-authentication for sensitive operations
