## Quick Setup

Enable core plugins in your AuthSome configuration:

<Tabs items={['YAML Configuration', 'Programmatic']}>
  <Tab value="YAML Configuration">
    ```yaml
    auth:
      plugins:
        # Username/password authentication
        username:
          minPasswordLength: 8
          maxPasswordLength: 128
          requireUppercase: true
          requireLowercase: true
          requireNumber: true
          requireSpecialChar: false
          allowUsernameLogin: true
          lockoutEnabled: true
          maxFailedAttempts: 5
          lockoutDuration: "15m"

        # Two-factor authentication
        twofa:
          totpIssuer: "MyApp"
          totpPeriod: 30
          totpDigits: 6
          backupCodeCount: 10
          backupCodeLength: 8
          trustedDeviceDays: 30

        # Anonymous authentication
        anonymous:
          enableAnonymous: true
          sessionExpiryHours: 72
          autoConvert: true

        # Email verification
        emailVerification:
          tokenLength: 32
          expiryHours: 24
          maxResendPerHour: 3
          autoSendOnSignup: true
          autoLoginAfterVerify: true

        # Bearer token middleware
        bearer:
          tokenPrefix: "Bearer"
          caseSensitive: false
    ```
  </Tab>
  <Tab value="Programmatic">
    ```go
    package main

    import (
        "log"

        "github.com/xraph/authsome"
        "github.com/xraph/authsome/plugins/username"
        "github.com/xraph/authsome/plugins/twofa"
        "github.com/xraph/authsome/plugins/anonymous"
        "github.com/xraph/authsome/plugins/emailverification"
        "github.com/xraph/authsome/plugins/bearer"
        "github.com/xraph/authsome/plugins/dashboard"
        "github.com/xraph/forge"
    )

    func main() {
        auth, err := authsome.New(config)
        if err != nil {
            log.Fatal(err)
        }

        // Register core plugins
        auth.RegisterPlugin(username.NewPlugin())
        auth.RegisterPlugin(twofa.NewPlugin())
        auth.RegisterPlugin(anonymous.NewPlugin())
        auth.RegisterPlugin(emailverification.NewPlugin())
        auth.RegisterPlugin(bearer.NewPlugin())
        auth.RegisterPlugin(dashboard.NewPlugin())

        // Initialize all plugins
        if err := auth.InitializePlugins(); err != nil {
            log.Fatal(err)
        }

        // Create Forge app and mount AuthSome
        app := forge.New()
        auth.Mount(app, "/auth")

        log.Println("Server starting on :8080")
        app.Listen(":8080")
    }
    ```
  </Tab>
</Tabs>

## Common API Endpoints

Core plugins provide these standard endpoints:

### Username Plugin
```http
POST /username/signup             # Register with username/password
POST /username/signin             # Sign in with username/password
```

### Two-Factor Authentication Plugin
```http
POST /2fa/enable                  # Enable 2FA for user
POST /2fa/verify                  # Verify 2FA code
POST /2fa/disable                 # Disable 2FA
POST /2fa/generate-backup-codes   # Generate new backup codes
POST /2fa/send-otp                # Send OTP via email/SMS
GET  /2fa/status                  # Get 2FA status
```

### Anonymous Plugin
```http
POST /anonymous/signin            # Create anonymous session
POST /anonymous/link              # Link anonymous account to registered account
```

### Email Verification Plugin
```http
GET  /email-verification/verify   # Verify email with token
POST /email-verification/resend   # Resend verification email
POST /email-verification/send     # Send verification email
GET  /email-verification/status   # Check verification status (authenticated)
```

## Security Considerations

<Callout type="warn">
**Security Best Practices**: Always use HTTPS in production, implement proper rate limiting, validate all inputs, and follow the security guidelines for each plugin.
</Callout>

### Username/Password Security
- Strong password policies with configurable requirements
- Password hashing with bcrypt/Argon2
- Account lockout after failed attempts (default: 5 attempts, 15 minute lockout)
- Password history to prevent reuse
- Optional password expiry

### Two-Factor Authentication Security
- Secure TOTP secret generation
- Backup codes for recovery
- Rate limiting on verification attempts
- Trusted device tracking

### Email Verification Security
- Cryptographically secure token generation
- Configurable token expiry (default: 24 hours)
- Rate limiting on resend (default: 3 per hour)
- Auto-send on signup for immediate verification

### Bearer Token Security
- Session-based token validation
- Configurable token prefix
- Session expiration checking
